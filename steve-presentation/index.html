The Stack

Eight purpose-built containers working in concert. Each service owns a single responsibility. The entire platform runs on a single homelab host.

gladius-api
The brain. FastAPI backend hosting the Claude audit agent, SSE streaming, parallel subagent architecture, and all API endpoints.
FASTAPI CLAUDE SONNET SSE 3,156 LOC
network-audit-mcp
MCP tool server. 10 tools including SSH, ChromaDB RAG, NVD CVE lookup, NMAP, Scapy, email, and audit result saving.
MCP PROTOCOL PARAMIKO SSH NMAP SCAPY
gladius-pyats
Automation Factory. LLM-driven pyATS/Genie script generation, Genie Learn/Diff, ITSM agent, scheduling engine, and Jira Cloud integration.
OLLAMA PYATS GENIE JIRA API
gladius-snmp
SNMP poller and protocol event engine. Monitors interfaces, BGP peers, and OSPF neighbors. Auto-creates Jira tickets on events.

Slack audit bot. Forwards messages to the Claude agent, collects results, posts formatted score cards with persistent conversation history.

What It Does

From a blank terminal to a full-stack network security operations platform. Every feature listed here was designed, built, and shipped during this project.

⚔
Agentic Security Audits
Claude SSHes into Cisco devices, runs show commands, scores configs against NIST 800-53 and CIS benchmarks, finds CVEs, and generates remediation reports.
⚡
Parallel Audit Engine
Device + Threat Intel subagents run concurrently. What took 87s sequentially now completes in under 40s. Auto-detects multi-device requests.
🔍
NMAP + Scapy Scanning
Agentic network scanning — Claude selects profiles and interprets results. Scapy packet forge for custom protocol testing. All via MCP tools.
📊
SNMP Monitoring
Real-time device polling with CPU, memory, interface tables, live utilisation sparklines, and protocol event detection (BGP, OSPF, interface state).
⚙
Automation Factory
Ollama generates pyATS/Genie test scripts from natural language. Execute, validate, view results. Genie Learn snapshots and AI-powered Diff analysis.
📅
Scheduling Engine
Cron-based scheduling for Learn and Diff jobs. Auto-runs, Slack/email notifications, severity-based Jira auto-ticketing.
🎫
Jira Cloud Integration
ITSM agent creates, queries, and closes Jira tickets. Auto-tickets from SNMP events. Duplicate detection with merge recommendations.
💬
ITSM Agent
Ollama-powered service desk agent. Drafts ITIL-aligned incident tickets, reasons about duplicate events, suggests ticket merges.
🛠
Tshoot Agent
Claude-powered diagnostics. SSHes into devices, pulls Jira context, updates tickets with findings, and closes resolved issues.
📨
Email Reports
Full HTML security reports generated in-browser and sent as email attachments. Claude's send_email calls are intercepted and templated.
🗨
Slack Integration
Two Slack bots: audit bot forwards requests to Claude, overseer has full project access and can proactively notify via DM.
📄
Intel Feeds
Live CVE data from NVD, Cisco PSIRT advisories, and EOX end-of-life lookups. Per-row "ASK" buttons for agentic deep-dives.

// 005 — BATTLES WON

Challenges & Solutions

Every ambitious project has its war stories. These are ours.

CHALLENGE
The audit_saved Race Condition
Audit results were being lost. The MCP tool POSTed to a webhook, but the SSE stream had already moved on. Reports never appeared in the dashboard.
SOLUTION
Emit from tool_input Directly
Bypassed the HTTP POST entirely. The SSE stream now emits audit_saved directly from the tool_input event, eliminating the race condition. Fixed on Day 1.
CHALLENGE
19-Second Tool Latency
Every MCP tool call took 9-19 seconds. The model was loading on every request. A full audit could take 5+ minutes.
SOLUTION
Persistent Sessions + Pre-Warm
Switched to persistent MCP sessions, set HF_HUB_OFFLINE=1, background init with pre-warm. Tool latency dropped to 1.5-3.5 seconds. Audit time cut 64%.
CHALLENGE
SNMP False Alarm Storm
Virtual interfaces (Null0, Loopback, SR0) appearing and disappearing between polls. BGP active↔idle cycling generating hundreds of alerts.
SOLUTION
FSM-Aware Filtering
Ignore list for virtual interfaces. 2-consecutive-miss requirement. BGP/OSPF alerts only when transitioning to/from established/full state. Noise eliminated.
CHALLENGE
Claude Refusing Scapy Commands
Claude's safety layer blocked SYN floods, ARP spoofing, and other offensive testing tools — even in an authorised homelab context.
SOLUTION
Authorization Context Prompt
Established explicit homelab authorization in the system prompt. Removed hidden packet count caps. Claude now executes security tests without pushback.
CHALLENGE
Ollama Ignoring Instructions
The small ITSM model (qwen2.5:7b) skipped duplicate ticket detection buried in long bullet lists. Merge suggestions never appeared.
SOLUTION
Prompt Architecture for Small Models
Moved critical instructions to dedicated sections at the END of the prompt with mandatory output formats and concrete examples. Small models need structure, not prose.
CHALLENGE
14,967-Line Single File
The entire frontend is one HTML file. CSS accidentally placed inside a JS template literal was silently ignored. Styles had no effect and we couldn't find why.
SOLUTION
Documented the Trap
generateReportHTML() contains a <style> block inside a template literal. CSS there is a string, not applied. Now documented in CLAUDE.md so the AI never makes this mistake again.
